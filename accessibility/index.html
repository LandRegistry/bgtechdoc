Accessibility Statement

Accessibility statement for Business Gateway Technical Documentation

This accessibility statement applies to the documentation at https://landregistry.github.io/bgtechdoc/.

This website is run by HM Land Registry. We want as many people as possible to be able to use this website. For example, that means you should be able to:

  • change colours, contrast levels and fonts
  • zoom in up to 400% without the text spilling off the screen
  • navigate most of the website using just a keyboard
  • navigate most of the website using speech recognition software
  • listen to most of the website using a screen reader (including the most recent versions of JAWS, NVDA, and Voiceover)

We’re also aiming to make the website text as simple as possible to understand.

Get advice from AbilityNet on making your device easier to use if you have a disability.

How accessible this website is

We know some parts of this website are not fully accessible:

  • A number of pages have arrows to indicate expanding a table
  • Some code segments are not focusable
  • A page has the word below to refer to tables
  • A page has partially incorrect heading styles
  • Some lists in tables are not formatted correctly on a page
  • There are issues caused by our Technical Documentation Template

Contact us if you need information on this website in a different format like:

  • accessible PDF
  • large print
  • easy read
  • audio recording
  • braille

Technical information about this website’s accessibility

HM Land Registry is committed to making its website accessible, in accordance with the Public Sector Bodies (Website and Mobile Applications)(No. 2) Accessibility Regulations 2018.

This website is partially compliant with the Web Content Accessibility Guidelines (WCAG) 2.2 (w3.org) AA standard, due to the non-compliances listed below.

Non-accessibility content

The content listed below is non-accessible for the following reasons.

WCAG 2.2 Accessibility Statement Updates

Non-compliance with the accessibility regulations

Some headings on “A Guide for Developers” page do not have the correct labels. This means assistive technologies will not read the content correctly. This fails WCAG 2.2 AA 2.4.6 (Headings and Labels).

Some labels for input search box missing. This fails WCAG 2.2 A 1.3.1 & 2.5.3 Label in name (Info and Relationships).

Some labels coded within the page will be shown missing. This fails WCAG 2.2 A 3.3.2 Labels or Instructions.

Also, some parts of this website are not fully accessible because of issues caused by our Technical Documentation Template.

All OpenAPI pages are inaccessible. These pages are controlled by a third party owned component.

If any issues arise, please contact our customer support team: https://landregistry.github.io/bgtechdoc/support/contact_us/

Content that’s not within the scope of the accessibility regulations

PDFs and other documents

The accessibility regulations do not require us to fix PDFs or other documents published before 23 September 2018 if they’re not essential to providing our services.

What we’re doing to improve accessibility

We are planning on reviewing and fixing the WCAG 2.2 non-compliant issues by October 31st, 2024.
